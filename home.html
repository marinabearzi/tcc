About

This web application was developed as a final paper for graduating in Computer Engineering at the Polytechnic School of University of Sao Paulo.

Our main goal here is to give the user the knowledge needed to build an application and guide it from the installation of the framework, through the creation of all the files and finalizing with the execution and analysis of the simulation.



The framework

SUMO (Simulation of Urban MObility) (http://sumo.dlr.de/wiki/) is an open source, highly portable, microscopic and continuous road traffic simulation package designed to handle large road networks.

OMNeT++ (https://omnetpp.org) is an extensible, modular, component-based C++ simulation library and framework, primarily for building network simulators.

Veins (http://veins.car2x.org/) is an open source framework for running vehicular network simulations. It is based on two well-established simulators: OMNeT++, an event-based network simulator, and SUMO, a road traffic simulator. It extends these to offer a comprehensive suite of models for IVC simulation.



The files

In order to build and run a simulation several files need to be created.

SUMO

.nod.xml is the file responsible for handling the nodes. All nodes have a location (x and y coordinate, describing distance to the origin in meters) and an id for future reference. Nodes can have types, for example, a junction controlled by a traffic light.

.edg.xml is the file responsible for handling the edges. Edges connect two nodes. There is a source node id, a target node id, and an edge id for future reference. Edges are directed, thus every vehicle travelling this edge will start at the node given in from and end at the node given in to.

.net.xml is the file responsible for linking .nod and .edg files and creating the road network. This is done by NETCONVERT, a command line application that comes with SUMO.

.rou.xml is the file responsible for creating cars and their routes. Vehicles have types defining their basic properties such as length, acceleration and deceleration, and maximum speed. A route consists of a sequence of edges previously defined. Finally, to create a single car just refer to the entries before and give it a departure time

.sumocfg is a configuration file responsible for gathering all the files above mentioned and for setting the simulation time and update interval.


OMNeT++

omnetpp.ini is a configuration file responsible for referencing the launchd.xml file and defining the simulation time and update interval of the simulation.

launchd.xml is the file responsible for linking SUMO and OMNeT++. In this file all SUMO files such as net.xml, rou.xml and .sumocfg are declared to be bound to OMNeT++. At each step of the simulation an instance of SUMO is created and connected with OMNeT++.

TraCICommandInterface is the file where data acquisition methods are implemented.

TraCIMobility is the file where methods of managing and updating the state of the vehicle are implemented.

  • SUMO 0.30.0

  3. Create .rou.xml file

Step 4: Create OMNeT++ Files

  1. Generate omnetpp.ini file
  2. Generate .launchd.xml file
  3. Generate TraCICommandInterface file
  4. Generate TraCIMobility file

Step 5: Move files to the right place

  1. Move all SUMO files as well as omnetpp.ini and .launchd.xml to
    2. .../veins-4.6/examples/veins

Step 6: Run your simulation

  1. Start the framework according to the tutorial on Step 2
  2. Build the project
  3. Run the simulation
